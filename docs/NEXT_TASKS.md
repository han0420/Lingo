# Lingo 后续任务清单

- 文档状态：Active
- 创建日期：2026-07-23
- 适用版本：0.1.x
- 目标：将当前“功能骨架可运行”的版本推进到“可稳定日常使用、可分发”的原生 macOS 应用

## 1. 当前状态

Lingo 已具备 SwiftUI 菜单栏应用、前台应用监听、规则匹配、Carbon 输入法切换、配置持久化、登录项、通知和基础设置界面。

当前版本已完成 P0 首轮迭代（本地化、设置布局、图标回退），但仍存在以下待办：

- 设置窗口在窄窗口/长文本下的手动布局验收，以及浅色/深色截图验收。
- 真实输入法切换、通知授权、登录启动等系统集成尚未完成全面验收。
- 构建产物仅使用本地 ad-hoc 签名，尚不具备公开分发条件。

## 2. 优先级说明

| 优先级 | 含义 | 处理原则 |
| --- | --- | --- |
| P0 | 阻塞正常使用或严重影响第一印象 | 下一轮首先完成 |
| P1 | 影响可靠性、可维护性或关键体验 | P0 完成后集中处理 |
| P2 | 发布、分发和长期产品能力 | 稳定版本前完成 |
| P3 | 增强功能 | 根据真实使用反馈排期 |

## 3. P0：修复可见问题与基础体验

### 3.1 修复本地化资源加载

- [x] 新增统一的本地化访问层，明确使用 `Bundle.module` 加载 Swift Package 资源。
- [x] 替换界面中依赖 `Bundle.main` 的 `Text("key")`、按钮标题、提示、通知和错误文案。
- [x] 修复带参数文案的格式化与语言切换行为。
- [x] 设置窗口标题显示为 `Lingo` 或正确的本地化标题，不得显示 key。
- [x] 编写测试，验证英文和简体中文的关键文案都能解析且结果不等于 key。
- [x] 修正 README 中“中英文界面已实现”的描述，只有验证通过后再标记完成。

验收标准：

- 中文系统下不出现 `tab.rules`、`input.english` 等裸 key。
- 英文系统下所有核心界面显示英文。
- 菜单栏、设置页、编辑弹窗、错误状态和通知使用同一套本地化机制。

### 3.2 重做设置窗口布局

- [x] 确定设置窗口合理的默认尺寸和最小尺寸，避免大面积无意义留白。
- [x] 将顶部导航调整为符合 macOS 设置应用习惯的工具栏或紧凑分栏。
- [x] 调整搜索、新增、选择应用按钮的宽度、层级和间距。
- [x] 为规则列表建立稳定列布局：应用信息、输入法、启用状态、操作。
- [x] 确保“中文/英文”完整显示，不使用被截断的紧凑 Picker。
- [x] 为启用开关和编辑按钮增加可访问标签与 tooltip。
- [x] 空列表、无搜索结果和加载失败使用不同的空状态文案。
- [x] 在窄窗口、长应用名和长 Bundle ID 下验证布局。

验收标准：

- 默认窗口打开后信息密度合理，无明显大片空白。
- 所有控件文字完整可读，行高和对齐保持一致。
- 界面在浅色和深色模式下均清晰可用。
- 键盘可以遍历主要交互控件。

### 3.3 完善应用图标和应用信息解析

- [x] 通过 Bundle ID 定位已安装应用，并缓存应用 URL 与图标。
- [x] 处理应用未安装、移动位置、Bundle ID 变化和 Spotlight 不可用场景。
- [x] 为无法找到图标的规则提供统一的 SF Symbol 回退图标。
- [x] 核对 VS Code、Cursor、微信等预设 Bundle ID 是否与实际发行版一致。
- [x] 避免在 SwiftUI 每次刷新行视图时重复访问 `NSWorkspace`。

验收标准：

- 已安装的常见应用正确显示原始图标。
- 找不到应用时不显示损坏文件图标，并明确标记“应用未找到”。

### 3.4 修复首次启动与设置入口

- [x] 确认首次启动是否自动打开设置窗口，并记录产品决策。
- [x] 确保点击菜单栏“设置”总能激活已有窗口，不重复创建多个设置窗口。
- [x] 关闭设置窗口后应用继续运行；再次打开时恢复合理位置和尺寸。
- [x] 菜单栏总开关状态与设置页保持实时同步。

验收标准：

- 菜单栏入口、窗口生命周期和后台驻留行为符合原生 macOS 预期。

## 4. P1：核心能力与可靠性

### 4.1 输入源发现与用户配置

- [x] 枚举本机所有可选择的键盘输入源，显示名称和 Source ID。
- [x] 不再仅依赖写死的搜狗、系统拼音、ABC、US 候选列表。
- [x] 允许用户分别选择“中文输入源”和“英文输入源”。
- [x] 候选输入源被卸载或禁用时提供清晰错误和重新选择入口。
- [x] 保留旧配置迁移逻辑，确保升级后已有规则不丢失。

验收标准：

- 用户可使用非搜狗的第三方中文输入法。
- 输入源不可用时保持当前输入法，不反复弹通知或产生切换循环。

### 4.2 强化切换状态机

- [x] 测试总开关关闭时不调用输入源服务。
- [x] 测试同一应用重复激活时不重复切换。
- [x] 明确 Lingo 自身成为前台应用时是否跳过切换。
- [x] 防止快速切换应用产生过期切换或状态竞争。
- [x] 记录最近一次成功切换的应用、输入源和时间，仅保留在内存中。
- [x] 区分“规则命中”“使用全局默认”“输入源不存在”和“系统切换失败”。

验收标准：

- 快速切换应用时，最终输入法始终对应当前前台应用。
- 相同前台应用不会触发重复通知或重复系统调用。

### 4.3 完善规则管理

- [ ] 新增规则时检测重复 Bundle ID，并引导编辑已有规则。
- [ ] 删除规则前提供符合 macOS 习惯的确认或可撤销机制。
- [ ] 支持按应用名和 Bundle ID 搜索，并保持大小写不敏感。
- [ ] 支持规则排序，至少提供应用名排序和启用状态分组。
- [ ] 区分预设规则、用户创建规则和已失效规则。
- [ ] 手动输入 Bundle ID 时提供格式校验和即时反馈。

验收标准：

- 无法保存空应用名、空 Bundle ID 或无意义重复规则。
- 搜索、编辑、启停和删除后列表与持久化结果一致。

### 4.4 系统集成验证

- [ ] 在系统拼音、搜狗拼音和 ABC 环境分别验证切换。
- [ ] 验证通知首次授权、拒绝授权和在系统设置中重新开启。
- [ ] 验证登录项注册、取消和重启 Mac 后自动运行。
- [ ] 验证睡眠唤醒、切换用户和输入源列表变化后的恢复。
- [ ] 验证应用没有辅助功能权限时仍可监听前台应用。
- [ ] 确认 Carbon TIS API 在目标最低版本 macOS 14 上行为一致。

### 4.5 扩充自动化测试

- [x] 为 `LingoStore` 注入 mock 输入源服务并测试副作用调用次数。
- [ ] 覆盖配置默认值、往返、损坏数据、旧版本迁移和无效数据。
- [ ] 覆盖规则去重时 ID、名称、启用状态和输入法的更新策略。
- [x] 增加本地化 key 对称性测试，确保中英文 key 集合一致。
- [x] 为应用信息解析增加不依赖真实安装环境的测试边界。

验收标准：

- 核心 Store、规则策略、配置迁移和本地化问题均有回归测试。
- `swift test` 能在干净环境稳定通过。

## 5. P2：工程化与发布准备

### 5.1 应用品牌与资源

- [x] 设计 Lingo 正式 App Icon，并生成 `.icns` 与所需尺寸。
- [x] 设计菜单栏 Template 图标，验证 Retina、深色和浅色模式。
- [ ] 补齐关于页、版本号、版权和隐私说明入口。
- [ ] 确认 Bundle ID、产品名、版本号和开发者身份，发布后不随意更换。

### 5.2 构建与签名

- [ ] 构建 Universal 2 产物，覆盖 Apple Silicon 和 Intel。
- [ ] 增加明确的 Debug、Release 构建配置。
- [ ] 使用 Developer ID Application 正式签名。
- [ ] 启用 Hardened Runtime，并确定最小 entitlements。
- [ ] 完成 Apple notarization 和 stapling。
- [ ] 打包 DMG 或 ZIP，并验证从下载、解压到首次启动的完整流程。
- [ ] 构建脚本禁止公开发布时静默回退到 ad-hoc 签名。

验收标准：

- `codesign --verify --deep --strict`、`spctl --assess` 和公证检查全部通过。
- 在一台未参与开发的 Mac 上可以正常安装、首次启动和卸载。

### 5.3 发布文档与安全

- [ ] 新增 `PRIVACY.md`，明确本地存储、前台应用信息和无网络行为。
- [ ] 新增安装、卸载、权限排查和输入源故障说明。
- [ ] 新增版本发布清单与回滚步骤。
- [ ] 安全检查改为基于 Git 可发布文件运行；项目初始化 Git 后启用 diff/status 检查。
- [ ] 确认配置和日志不保存用户完整应用使用历史。

### 5.4 持续集成

- [ ] 初始化并规范 Git 仓库与 `.gitignore`。
- [ ] CI 执行 `swift test`、Release 编译、安全检查和 skill 校验。
- [ ] 对本地化缺失、格式错误和未更新 spec 增加自动检查。
- [ ] 建立版本号和 changelog 规则。

## 6. P3：可选增强功能

- [ ] 支持导入和导出非敏感规则配置。
- [ ] 支持为应用临时暂停切换一段时间。
- [ ] 支持在菜单栏快速查看当前前台应用及命中规则。
- [ ] 支持更多规则条件，例如无规则时保持当前输入法，而不是使用全局默认。
- [ ] 支持规则分组和批量启停。
- [ ] 支持快捷键暂停或恢复自动切换。
- [ ] 在不记录长期历史的前提下提供本次运行的诊断信息。

这些增强项必须基于真实用户反馈进入 spec，不应阻塞 P0/P1 稳定性工作。

## 7. 建议实施顺序

```text
本地化加载
  ↓
设置窗口与规则列表重构
  ↓
应用图标/信息解析
  ↓
输入源发现与选择
  ↓
Store 状态机与自动化测试
  ↓
通知、登录项、真实设备验收
  ↓
图标、签名、公证与分发
```

建议每个阶段单独建立 `docs/specs/<feature-name>.md`，按 RED → GREEN → REFACTOR 推进，不把全部任务放进一次大改动。

## 9. 完成定义

单项任务只有同时满足以下条件才可勾选：

- 对应 spec 已更新并标明实现状态。
- 自动化测试通过，且新增行为有回归覆盖。
- 英文和简体中文资源同步。
- `swift test` 与 `script/security_check.sh` 通过。
- 涉及 UI 或系统集成时完成相应手动检查。
- README、架构和用户文档与实际行为一致。
- 未把真实用户名、绝对路径、应用使用历史、签名材料或其他隐私数据写入仓库。
