# Spec-first 开发流程

新增功能或用户可见行为修改必须依次执行：

1. 阅读 `AGENTS.md`、架构、相关 spec、实现与测试。
2. 在 `docs/specs/<feature>.md` 写明状态、目标、非目标、需求、边界、验收、技术方案、测试与未决问题。
3. 对清晰需求记录最小假设；会改变隐私、安全或核心 UX 的选择先与用户确认。
4. 先写测试并运行，确认因目标行为缺失而 RED。
5. 写最小实现使测试 GREEN，再保持全绿重构。
6. 运行 `swift test`、安全检查和与改动风险相称的应用启动验证。
7. 将 spec 标为 `Implemented`，补实现映射和验证结果，并同步 README/架构/本地化。

无法自动测试的真实输入法、菜单栏、通知和登录项交互要在 spec 中保留可重复的手动检查。
