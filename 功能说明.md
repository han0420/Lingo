# Lingo — 功能说明

> **应用名：Lingo**（原「输入法切换器 / switch-input」）
>
> 取名释义：Lingo 意为「语言/俗语」，呼应本应用的核心——让输入法（语言）跟随当前应用自动切换。
>
> 本文档为产品功能描述，供后续重新实现为 Mac 原生应用时作为需求参考，不含技术实现细节。

---

## 一、它解决什么问题

在 Mac 上，不同应用往往需要不同的输入法：

- 聊天软件（微信、钉钉、飞书）→ 要用**中文**
- 写代码、用终端、填英文表单 → 要用**英文**

但 macOS 不会记住「每个应用该用什么输入法」，每次切换应用都要手动切输入法。一旦忘了切，打字时才发现输入法不对，再手忙脚乱切回去——一天要重复几十次，非常打断思路。

**Lingo 让输入法「跟着应用走」：切到哪个应用，输入法就自动变成你为它设好的状态，从此不用再手动切。**

---

## 二、核心功能

### 1. 按应用自动切换输入法（核心能力）

- 切换到某个应用时，自动把输入法切到你为它预设的状态（中文 / 英文）
- 例：切到微信 → 自动变中文；切到 VSCode → 自动变英文
- 没有专门设置的应用，统一走「全局默认输入法」
- 整个过程无感、即时，用户无需任何操作

### 2. 应用规则管理

为每个应用单独配置默认输入法：

- **新增规则**：可直接从系统已安装应用列表里选，也可手动填入应用
- **编辑 / 删除**：随时修改某个应用的输入法设定，或删除规则
- **单独开关**：每条规则可独立启用 / 禁用（临时关掉某个应用不切，不影响其他规则）
- **搜索**：在规则列表里按应用名快速查找
- **开箱即用**：内置常用应用的默认规则，装好即生效，无需逐个配置

### 3. 全局开关与默认设置

- **一键总开关**：临时关闭整个自动切换功能，不影响已保存的配置（再开即恢复）
- **全局默认输入法**：没配规则的应用统一用这个（一般设为英文）
- **开机自启动**：开机后自动运行，只在菜单栏待命，不弹窗打扰
- **切换通知**（可选）：每次自动切换时是否弹一个小通知，方便确认

### 4. 菜单栏常驻

- 一直待在屏幕顶部菜单栏，不占用 Dock 位置
- 点菜单栏图标可快速：开关功能、打开设置、退出应用
- 图标自动适配深色 / 浅色模式

---

## 三、使用流程

1. **首次启动**：按系统提示授权（应用需要读取当前前台应用信息）
2. **配置规则**：打开设置 → 在「应用规则」里添加常用应用，给每个应用选中文或英文
3. **日常使用**：正常用电脑即可，切应用时输入法自动变对，无需再管

---

## 四、开箱即用的预设规则

| 应用 | 默认输入法 |
|------|-----------|
| 微信 | 中文 |
| Safari | 英文 |
| Google Chrome | 英文 |
| Visual Studio Code | 英文 |
| Cursor | 英文 |

用户可在此基础上自由增删改。

---

## 五、支持的输入法

- 搜狗拼音输入法（中文）
- 系统自带 ABC 英文输入法
- 系统自带简体拼音输入法

---

## 六、一句话总结

**Lingo：一个常驻菜单栏的小工具，让你切到不同应用时，输入法自动变成你想要的状态——不用再手动切，打字永远是对的。**
