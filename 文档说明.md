## 二、项目简介

**switch-input** 解决的是 macOS 用户的一个高频痛点：在不同应用间切换时，输入法状态经常「错位」—— 比如从微信（中文聊天）切到 VSCode（写代码）时还在中文输入法，每次都要手动切回英文。

本应用常驻菜单栏，后台监控当前前台应用，根据用户为每个应用预设的规则，自动把输入法切到中文或英文，做到「切到哪个 App，就是哪个输入法」。

## 三、核心功能

### 1. 自动切换输入法（核心）
- 切换前台应用时，自动按规则切换输入法
- 有规则的应用 → 切到规则指定输入法
- 无规则的应用 → 切到全局默认输入法

### 2. 应用规则管理
- 为每个应用（按 Bundle ID）配置默认输入法（中文 / 英文）
- 支持新增、编辑、删除、启用 / 禁用单条规则
- 支持搜索过滤
- 内置常用应用预设（微信→中文、Safari/Chrome/VSCode/Cursor→英文）
- 可从系统已安装应用列表中选择，也支持手动添加

### 3. 全局设置
- 启用 / 禁用自动切换（一键开关）
- 全局默认输入法（未配置规则的应用使用）
- 开机自启动（启动后仅显示菜单栏图标，不弹窗）
- 切换通知开关

### 4. macOS 原生体验
- 菜单栏（Tray）常驻图标，适配深浅模式（Template 图标 + Retina @2x）
- 菜单栏右键菜单：启用状态切换、打开设置、退出
- macOS 隐藏式标题栏（红绿灯按钮自定义位置）
- 关闭窗口 = 隐藏（不退出），保持后台运行

## 五、核心工作原理

### 1. 应用切换监控（app-monitor.ts）

采用 **定时轮询** 方案，而非事件订阅（macOS 无公开的前台应用切换事件 API）：

- 监控间隔：**500ms**
- 通过 `osascript` 执行 AppleScript，调用 `System Events` 获取前台应用的 Bundle ID
- 与上次记录的 Bundle ID 对比，发生变化才触发切换逻辑
- 流程：`获取当前 Bundle ID → 对比变化 → 查规则 → 切输入法`

```
monitorLoop (每 500ms)
    │
    ├─ osascript 获取当前前台应用 bundleId
    │
    ├─ bundleId 变了？ ─ 否 → 跳过
    │
    └─ 是 → handleAppSwitch(bundleId)
            ├─ 功能未启用 → return
            ├─ 查找规则 (getAppRuleByBundleId)
            │   ├─ 有规则 → switchInputMethod(rule.inputMethod)
            │   └─ 无规则 → switchInputMethod(全局默认)
```

### 2. 输入法切换（ime-switcher.ts）— 关键实现

这是本项目最有技术含量的部分。macOS 没有公开的命令行输入法切换工具，本项目的方案是：

**运行时动态生成一个 Swift 脚本（ime-helper），通过 Carbon API 切换输入法。**

- Swift helper 脚本首次运行时被写入 `~/Library/Application Support/switch-input/ime-helper`
- 脚本调用 Carbon 框架的 `TISSelectInputSource` / `TISCopyCurrentKeyboardInputSource` / `TISCreateInputSourceList`
- 支持三种调用方式：
  - 无参数 → 返回当前输入法 Source ID
  - `--list` → 列出所有可选输入法
  - `<sourceId>` → 切换到指定输入法
- 切换逻辑带**主备回退**：先试主输入法，失败再试备选输入法

支持的输入法及 Source ID：

| 输入法 | Source ID | 用途 |
|--------|-----------|------|
| 搜狗拼音 | `com.sogou.inputmethod.sogou.pinyin` | 中文（主） |
| 系统简体拼音 | `com.apple.inputmethod.SCIM.ITABC` | 中文（备选） |
| 系统 ABC | `com.apple.keylayout.ABC` | 英文（主） |
| 美国英文 | `com.apple.keylayout.US` | 英文（备选） |

### 3. 配置存储（config-store.ts）

- 使用 `electron-store` 持久化
- 配置文件位置：`~/Library/Application Support/switch-input/switch-input-config.json`
- 数据结构：
  - `globalConfig`：全局配置（启用状态、默认输入法、开机自启、通知开关）
  - `appRules`：应用规则数组（bundleId、appName、inputMethod、enabled）

### 4. 应用列表获取（app-list.ts）

- 用 `mdfind`（Spotlight）扫描 `/Applications`、`~/Applications`、`/System/Applications`
- 用 `mdls` 获取每个应用的 Bundle ID
- 图标获取：优先用 Electron 原生 `app.getFileIcon`，转 Base64 传给前端
- 内置常用应用预设列表（微信、QQ、钉钉、飞书、各浏览器、各开发工具等）

### 5. IPC 通信

所有通道名集中在 `src/types/index.ts` 的 `IPC_CHANNELS` 常量：
- 主进程用 `ipcMain.handle()` 处理异步请求
- 渲染进程通过 `window.electronAPI.*` 调用（preload 用 contextBridge 安全暴露）

## 六、权限要求

| 权限 | 是否需要 | 说明 |
|------|----------|------|
| 辅助功能权限 | 可能需要 | 通过 AppleScript 访问 `System Events` 获取前台应用信息，部分 macOS 版本会要求授权 |
| Swift 运行时 | 自带 | macOS 系统自带，无需额外安装 |
| 输入法切换权限 | 无需 | Carbon API 直接调用，无需特殊授权 |

> 注：README 中提到需授予辅助功能权限，CLAUDE.md 中说明输入法切换本身无需特殊权限（因使用 Carbon API）。两者并不冲突——辅助功能权限主要针对应用监控环节。

## 九、总结

switch-input 是一个**聚焦单一痛点、实现完整**的 macOS 实用工具：

- **定位清晰**：只做一件事——按应用自动切换输入法，做透
- **技术方案务实**：用 Swift + Carbon API 解决 macOS 无公开输入法切换命令的问题，是本项目的核心亮点
- **架构规范**：Electron 主 / preload / 渲染三层清晰分离，TypeScript 全量类型，IPC 通道集中定义
- **体验原生**：菜单栏常驻、Template 图标、深浅模式适配、关窗即藏

适合作为「Electron + Swift 混合调用 macOS 系统 API」的参考实现。
